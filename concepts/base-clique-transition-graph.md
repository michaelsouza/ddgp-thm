## Intuition

The base-clique transition graph is a derived graph whose vertices are the predecessor cliques used to discretize later vertices.

For a DDGP instance, each non-seed vertex $v_i$ is placed from a predecessor set $U_i$. This set is a clique and acts as the local base from which $v_i$ is constructed. Instead of looking directly at vertices and dependency arcs, the base-clique transition graph looks at how these local bases evolve.

This view is useful for experiments on solution counts because it separates two effects:

- which base clique creates a new vertex;
- whether that new vertex is later reused by another base clique.

It can reveal local branching structure that is less visible in the ordinary [discretization dependency set](discretization-dependency-set.md) graph.

The same structure also defines [base generators](base-generator.md), which toggle
all local decision vertices generated by a base clique together with their
dependency descendants.

## Definition

Let $G = (V, E, d)$ be a DDGP instance in $\mathbb{R}^K$ with discretization order $v_1,\dots,v_n$ and predecessor sets $U_i$ for $i > K+1$.

The vertices of the base-clique transition graph are the distinct predecessor cliques

$$
\mathcal{C}_U
= \{U_i \mid i > K+1\}.
$$

For each clique $C \in \mathcal{C}_U$, define the set of vertices generated from $C$ as

$$
\operatorname{Gen}(C)
=
\{v_i \mid i > K+1,\ U_i = C\}.
$$

A directed transition edge is added from $C$ to $D$ when a vertex generated from $C$ is later used as a predecessor in $D$:

$$
(C,D) \in A_{\mathcal{C}}
\quad\Longleftrightarrow\quad
C \ne D
\text{ and }
\operatorname{Gen}(C) \cap D \ne \emptyset.
$$

Thus, a transition edge means that a vertex created by one base clique becomes part of another base clique.

## Overlap edges

For visualization, it is also useful to add undirected overlap edges. These are not dependency arcs; they only mark that two base cliques have a large intersection:

$$
\{C,D\} \in O_{\mathcal{C}}
\quad\Longleftrightarrow\quad
C \ne D
\text{ and }
|C \cap D| \ge K-1.
$$

In dimension $K=2$, this means two base edges share at least one endpoint. In higher dimensions, it identifies bases that differ by a small local replacement.

## Levels

A natural level for a base clique is inherited from the vertex dependency levels:

$$
\ell(C) = \max_{v \in C} \ell(v).
$$

With this convention, seed-only base cliques have level zero, and a transition edge usually moves from a lower or equal level to a higher level.

## Use with pruning edges

Given a [pruning edge](pruning-edge.md) $\{u,v\}$, the relevant part of the base-clique transition graph is often the subgraph induced by cliques contained in

$$
\operatorname{Fix}_U(u) \cup \operatorname{Fix}_U(v).
$$

Cliques that contain either $u$ or $v$ are especially important because they show where the endpoints of the pruning edge become part of later discretization bases.
