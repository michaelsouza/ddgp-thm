### Issue 1: Abstract Is Too Jargon-Dense For First Contact

* **Severity**: Major
* **File & Line**: `article/main.tex` (Line 23)
* **Tsitsiklis Basis**: Section 3.3: the abstract should be crisp, declarative, and light on notation; a usual structure is "We consider..." followed by "We show that...."
* **Problem**: The abstract introduces DDGP, DMDGP, cone/base generators, mirror-labeled violations, zero-violation shifts, and rank-nullity before giving a clean problem-result-count path. It also uses `DDGP` before explicitly introducing the acronym.
* **Original LaTeX**:

```latex
The Discretizable Distance Geometry Problem admits binary Branch-and-Prune
representations, but non-consecutive predecessor sets obstruct global
combinatorial solution counts. We develop a local rank-count theory for
generic DDGP active subproblems. The construction associates graph-derived cone
and base generators with branch masks and records mirror-labeled edge
violations over \(\mathbb{F}_2\). For any reference solution, the feasible local
branch codes are exactly an affine space generated by the zero-violation
shifts. Thus the generic local solution count follows from rank-nullity,
without enumerating Branch-and-Prune branches, and specializes to the known
clique-based DMDGP and Combinatorial DDGP counts.
```

* **Recommended Replacement**:

```latex
In the Discretizable Distance Geometry Problem (DDGP), nonconsecutive
predecessor sets prevent a global solution count that depends only on graph
topology. We consider the local counting problem obtained by fixing an active
set of pruning constraints in a generic DDGP subproblem. We show that, relative
to any reference solution, the feasible binary Branch-and-Prune codes form an
affine space over \(\mathbb{F}_2\). Its direction is determined by the branch
shifts that produce no labeled pruning-edge violation. Consequently, the local
solution count follows from a rank computation, without enumerating
Branch-and-Prune branches. The formula recovers the known clique-based counts
for the Discretizable Molecular Distance Geometry Problem and for
Combinatorial DDGP instances.
```

* **Rationale**: The replacement preserves the claims but orders them as object, scope, result, and consequence.

### Issue 10: Preservation Proof Uses An Informal Noncommutation Argument

* **Severity**: Major
* **File & Line**: `article/sections/results.tex` (Line 65)
* **Tsitsiklis Basis**: Section 5: proofs should have a forward, nearly linear structure and should distinguish rigorous argument from explanation.
* **Problem**: The proof acknowledges that reflections generally do not commute, then asserts preservation of coordinates and later mirrors without a clear induction step.
* **Original LaTeX**:

```latex
Reflection through the affine span $\operatorname{aff}(C)$ preserves edges with both endpoints moved, edges with neither endpoint moved, and crossing edges whose fixed endpoint lies directly on the mirror. Although spatial reflections generally do not commute, here any active edge crossing the block support $S_C$ has its fixed endpoint lying directly on the mirror $\operatorname{aff}(C)$. Consequently, each block reflection acts symmetrically on active edges, preserving the coordinates and affine hyperplanes representing subsequent mirror cliques. The composition of these reflections is therefore well-defined, and applying these block reflections one mirror clique at a time preserves every edge in $F$. Their net branch mask is $\bigoplus_C S_C = M\alpha = h$. Thus, $s \oplus h$ realizes every active edge in $F$.
```

* **Recommended Replacement**:

```latex
Apply the nonzero blocks \((S_C,C)\) one at a time and argue by induction on the number of applied blocks. At the step for \(C\), vertices of \(C\) are not toggled by \(S_C\), so they lie on the current mirror \(\operatorname{aff}(C)\). For any edge \(e\in F\), either both endpoints are reflected, neither endpoint is reflected, or exactly one endpoint is reflected. In the first two cases the current length of \(e\) is preserved. In the crossing case, the preceding paragraph shows that the fixed endpoint lies in \(C\), hence on \(\operatorname{aff}(C)\), so the length of \(e\) is again preserved. Thus each block preserves all active-edge lengths at the moment it is applied. After all blocks are applied, the accumulated branch mask is \(\bigoplus_C S_C=M\alpha=h\), and therefore \(s\oplus h\in\Xi_F\).
```

* **Rationale**: This replacement gives the reader a checkable forward proof and avoids leaning on an unsupported commutation claim.
