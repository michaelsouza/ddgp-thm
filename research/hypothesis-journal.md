# DDGP Local Solution Count Hypothesis Journal

## Purpose

This journal tracks competing hypotheses for estimating the number of local DDGP solutions associated with one or more pruning edges.

The goal is not to record only successful ideas. The goal is to preserve the search path: hypotheses, tests, failures, refinements, and new concepts that may lead to a theorem.

## Current conventions

- The first \(K+1\) vertices are fixed as the initial rigid simplex.
- Binary decisions start at vertex \(K+2\).
- For a pruning edge \(e=\{u,v\}\), the local vertex set is
  \[
  L_e=\operatorname{Fix}_U(u)\cup \operatorname{Fix}_U(v).
  \]
- For a pruning-edge set \(P\), the local vertex set is
  \[
  L_P=\bigcup_{\{u,v\}\in P}\left(\operatorname{Fix}_U(u)\cup \operatorname{Fix}_U(v)\right).
  \]
- The local base count is computed on the subgraph induced by \(L_e\) using the base edges.
- The local valid count is computed on the same local subgraph after adding \(e\).
- The local solution code is the set of binary branch strings representing the local realizations.

## Current lead

The strongest empirical pattern so far is:

> The local base solution code and the local valid solution code are affine subspaces of the local binary branch space.

Across the 1530 one-pruning-edge instances tested so far, including 120 generic \(K=3\) instances:

| Test | Result |
|---|---:|
| Base code is affine | 1530/1530 |
| Valid code is affine | 1530/1530 |
| Full XOR stabilizer predicts valid count | 1530/1530 |
| Valid code is one orbit of its full XOR stabilizer | 1530/1530 |
| Graph-mask span recovers valid stabilizer rank | 1530/1530 |
| Mirror-compatible graph masks predict valid count | 1529/1530 |
| Mirror-compatible graph masks predict valid count on globally nondegenerate stress tests | 570/570 |

Across the 775 multi-pruning-edge generic instances:

| Test | Result |
|---|---:|
| Valid code is affine | 775/775 |
| Full XOR stabilizer predicts valid count | 775/775 |
| Labelled violation kernel predicts valid count | 775/775 |
| Full valid stabilizer lies in labelled violation kernel span | 775/775 |

The rank-only counting script `scripts/ddgp_rank_count.py` tests the algorithmic formula

\[
r_P=
\operatorname{rank}
\begin{bmatrix}
M\\
V
\end{bmatrix}
-
\operatorname{rank}(V),
\qquad
|\Xi_P|=2^{r_P},
\]

without enumerating kernel masks. With enumeration enabled only as verification, it matched exact local valid counts in 2225/2225 generic DDGP/DMDGP tests across \(K=2\), \(K=3\), one pruning edge, and multiple pruning edges.

This suggests the local count may be governed by a stabilizer subgroup:

\[
|\Xi_e| = 2^{\dim \operatorname{Stab}(\Xi_e)}.
\]

The open problem is to characterize \(\operatorname{Stab}(\Xi_e)\) from the DDGP graph structure, without enumerating \(\Xi_e\).

## Hypotheses

### H1: Local Affine Solution Code Hypothesis

Status: most promising.

Claim: For generic DDGP instances, the local base solution code and the local valid solution code are affine subspaces of the binary branch space over \(\mathbb{F}_2\).

Motivation: DMDGP solution sets are generated by partial-reflection groups. The DDGP does not preserve the same linear tail structure, but the local binary solution sets may still be affine after restricting to \(L_e\).

Test: Enumerate local base and valid bitstrings. Compute the affine hull rank and check whether

\[
|\Xi| = 2^{\operatorname{rank}(\operatorname{affine\ hull}(\Xi))}.
\]

Result: Passed on 1530/1530 tested single-pruning instances, including 120 generic \(K=3\) instances.

Decision: Keep as the primary conjecture. The next step is to prove or falsify it on larger and higher-dimensional datasets.

### H2: Full XOR Stabilizer Count Hypothesis

Status: most promising, but currently empirical.

Claim: The local valid count equals the size of the full XOR stabilizer of the local valid solution code:

\[
|\Xi_e| = |\operatorname{Stab}(\Xi_e)|.
\]

Equivalently,

\[
|\Xi_e| = 2^{\dim \operatorname{Stab}(\Xi_e)}.
\]

Motivation: If \(\Xi_e\) is an affine subspace, it is a single coset of its stabilizer.

Test: Compute all XOR masks \(g\) such that \(\Xi_e\oplus g=\Xi_e\), then compare \(2^{\operatorname{rank}}\) with the enumerated local valid count.

Result: Passed on 1530/1530 tested single-pruning instances, including 120 generic \(K=3\) instances.

Decision: Keep. This is currently the cleanest algebraic formulation, but it is not yet graph-theoretic because the stabilizer is computed from the enumerated solution code.

### H3: Endpoint-Separating Cone Rank Hypothesis

Status: useful approximation, not sufficient.

Claim: Each allowed dependency-cone toggle that separates the pruning endpoints removes one binary degree of freedom:

\[
\log_2 |\Xi_e|
=
\log_2 |\Xi_0|
-
\operatorname{rank}(\mathcal{S}_e),
\]

where \(\mathcal{S}_e\) is the family of allowed endpoint-separating cone masks.

Motivation: In the DMDGP, a pruning edge kills partial reflections whose levels lie between the endpoints. In the DDGP, endpoint-separating cones are the natural replacement for linear intervals.

Test: For each local decision vertex \(w\), compute its forward dependency cone. If the cone contains exactly one endpoint of \(e\), test whether the corresponding cone mask preserves the local base code and whether it is killed by \(e\).

Result:

| Dataset | Matches |
|---|---:|
| 60 baseline | 53/60 |
| 300 n=8 | 270/300 |
| 150 n=9 | 141/150 |
| 300 n=10 | 287/300 |
| 150 n=11 | 144/150 |
| 300 n=10 generic | 287/300 |
| 150 n=11 generic | 139/150 |

Decision: Keep as a structural feature, but reject as a complete theory. Some valid stabilizer masks are not simple dependency-cone masks.

### H4: Cone-Preserving Stabilizer Hypothesis

Status: rejected as complete theory.

Claim: The valid solution count is determined by the rank of cone masks that preserve the valid code.

Test: Restrict the stabilizer computation to dependency-cone masks only.

Result:

| Dataset | Matches |
|---|---:|
| 60 baseline | 52/60 |
| 300 n=8 | 268/300 |
| 150 n=9 | 139/150 |
| 300 n=10 | 278/300 |
| 150 n=11 | 142/150 |
| 300 n=10 generic | 286/300 |
| 150 n=11 generic | 133/150 |

Decision: Reject as a complete theory. The failures show that some valid symmetries are XOR combinations not represented by individual dependency cones.

### H5: Base-Clique Transition Graph Scalar-Metric Hypothesis

Status: weak evidence.

Claim: Simple scalar metrics of the base-clique transition graph, such as number of transitions, overlap edges, paths, or sources/sinks, correlate directly with the number of local solutions.

Motivation: The base-clique transition graph records how local bases evolve and how generated vertices are reused.

Test: Compute local base-clique counts, transition counts, overlap counts, and path counts.

Result: No strong direct correlation was observed in the baseline data. These metrics are likely too coarse.

Decision: Do not discard the base-clique transition graph. Instead, move toward labelled reachability and endpoint signatures.

### H6: Labelled Base-Clique Stabilizer Hypothesis

Status: strongest graph-theoretic lead so far.

Claim: The full XOR stabilizer can be reconstructed from a labelled version of the base-clique transition graph, where each transition records which generated vertex is reused.

Motivation: Unlabelled adjacency powers lose the identity of the vertex carrying dependency. The failures of H3 and H4 suggest that stabilizer masks may be combinations of several cone effects.

First refined construction:

- Keep the individual dependency-cone masks.
- Add one base-clique closure mask \(m_C\) for each predecessor clique \(C\), where \(m_C\) toggles all local decision vertices generated by \(C\) and their dependency descendants.
- Let \(\mathcal{G}_e\) be the graph-derived mask family consisting of cone masks and base-clique closure masks that preserve the local base code.
- Test whether the valid stabilizer is contained in \(\operatorname{span}_{\mathbb{F}_2}(\mathcal{G}_e)\), and whether the rank of the valid part of this span equals \(\log_2 |\Xi_e|\).

Second refined construction:

Each graph mask is treated as a pair \((m,C)\), where \(m\) is the binary support and \(C\) is the mirror clique.

For an edge set \(F\), \((m,C)\) is mirror-compatible with \(F\) if every edge crossing the support boundary has its fixed endpoint in \(C\). Equivalently, for every \(\{a,b\}\in F\),

\[
|\{a,b\}\cap \operatorname{supp}(m)|=1
\Longrightarrow
\{a,b\}\setminus \operatorname{supp}(m) \subseteq C.
\]

The predicted local valid count is

\[
\widehat{|\Xi_e|}
=
2^{\operatorname{rank}(\mathcal{M}_e)},
\]

where \(\mathcal{M}_e\) is the family of cone masks and base-clique closure masks that are mirror-compatible with both the local base edge set and the pruning edge \(e\).

Result:

| Dataset | Graph span recovers valid stabilizer rank | Mirror-compatible masks predict valid count | Full valid stabilizer in mirror-compatible span |
|---|---:|---:|---:|
| 60 baseline | 60/60 | 60/60 | 60/60 |
| 300 n=8 | 300/300 | 300/300 | 300/300 |
| 150 n=9 | 150/150 | 150/150 | 150/150 |
| 300 n=10 | 300/300 | 299/300 | 299/300 |
| 150 n=11 | 150/150 | 150/150 | 150/150 |
| 300 n=10 generic | 300/300 | 300/300 | 300/300 |
| 150 n=11 generic | 150/150 | 150/150 | 150/150 |
| 120 K=3 n=9 generic | 120/120 | 120/120 | 120/120 |

The same graph-mask family recovered the full base stabilizer in 1529/1530 cases.

The only mirror-compatible failure so far is `ddgp-nonlast-nonseed-0054` in the non-filtered \(n=10\) dataset. It is caused by a near-collinearity: endpoint \(v_5\) of the pruning edge lies almost on the mirror line through \(\{v_1,v_2\}\), with distance approximately \(3.4\cdot 10^{-6}\). This is a nongeneric geometric coincidence, not a combinatorial pattern. After adding a global nondegeneracy filter that rejects nearly collinear triples, the mirror-compatible count passed 450/450 new stress tests.

Interpretation: H6 fixes the main weakness of H3 and H4. In examples such as `ddgp-last-nonseed-0002`, the valid stabilizer contains a mask like \(\{4,5,8\}\), which is not an individual cone but is generated by a base-clique closure mask.

Decision: Keep as the main graph-structural lead. The current candidate formula for one pruning edge is:

\[
|\Xi_e| =
2^{\operatorname{rank}(\mathcal{M}_e)}.
\]

The next unresolved step is to prove why this formula survives generic instances, multiple pruning edges, and dimensions beyond the currently tested \(K=2,3\).

### H7: Labelled Violation Kernel Hypothesis

Status: strongest lead for multiple pruning edges.

Claim: For a pruning-edge set \(P\), the valid local stabilizer is generated by the projection of a labelled violation kernel.

Construction:

- Start with graph-derived masks \((m_g,C_g)\) that are mirror-compatible with the local base edge set.
- For each pruning edge \(e\in P\), mark whether \(g\) violates \(e\).
- Label each violation by the pair \((e,C_g)\), not by \(e\) alone.
- Form the linear map

\[
\nu:\mathbb{F}_2^{\mathcal{G}}\to\mathbb{F}_2^{\mathcal{L}},
\]

where \(\mathcal{L}\) is the set of labelled violations.
- Take the kernel \(\ker\nu\), then project each generator combination to its binary mask support.

The predicted local valid count is

\[
|\Xi_P|
=
2^{\operatorname{rank}(\mathcal{K}_P)},
\]

where \(\mathcal{K}_P\) is the projected mask space from \(\ker\nu\).

Motivation: Multiple pruning edges can allow cancellation between generators that are not individually compatible with all pruning edges. Plain violation parity is too weak because it allows cancellations across different mirror cliques. Labelling by \((e,C)\) keeps the geometric type of the violation.

Result:

| Dataset | Pruning edges | Labelled violation kernel predicts count | Valid code affine |
|---|---:|---:|---:|
| `multi-pruning-instances-n8-generic` | 2, 3 | 200/200 | 200/200 |
| `multi-pruning-instances-n10-generic` | 2, 3 | 200/200 | 200/200 |
| `multi-pruning-instances-n9-p234-generic` | 2, 3, 4 | 225/225 | 225/225 |
| `multi-pruning-instances-n11-p234-generic` | 2, 3, 4 | 150/150 | 150/150 |
| `multi-pruning-instances-k3-n9-p23-generic` | 2, 3 | 120/120 | 120/120 |
| `multi-pruning-instances-k3-n10-p234-generic` | 2, 3, 4 | 120/120 | 120/120 |
| `multi-pruning-instances-dmdgp-k2-n10-p234-generic` | 2, 3, 4 | 120/120 | 120/120 |
| `multi-pruning-instances-dmdgp-k3-n10-p234-generic` | 2, 3, 4 | 120/120 | 120/120 |
| `multi-pruning-instances-k2-n12-p2345-generic` | 2, 3, 4, 5 | 200/200 | 200/200 |
| `multi-pruning-instances-k3-n12-p2345-generic` | 2, 3, 4, 5 | 200/200 | 200/200 |

The same rank-only algorithm also matched the \(K=3\) single-pruning dataset `hypothesis-instances-k3-n9-generic` in 120/120 cases.

Decision: Keep as the main multi-pruning conjecture. This is now the clearest route to an SBBU-like DDGP counting algorithm: build graph masks, build the labelled violation matrix, and compute a rank.

Algorithmic refinement: If \(M\) is the mask matrix and \(V\) is the labelled violation matrix, then the projected kernel rank is

\[
\operatorname{rank}(\mathcal{K}_P)
=
\operatorname{rank}
\begin{bmatrix}
M\\
V
\end{bmatrix}
-
\operatorname{rank}(V).
\]

This avoids enumerating the kernel or the local solution code. It is implemented in `scripts/ddgp_rank_count.py` and documented as [DDGP Labelled-Violation Rank Count](../concepts/ddgp-labelled-violation-rank-count.md).

Important caution: the rank of base-compatible graph generators should not be interpreted as a universal formula for the local base count on arbitrary multi-edge local unions. In the rank-count stress tests, the valid count matched 2225/2225 generic cases, while the auxiliary base-count prediction matched 2222/2225. The valid rank formula remains the main object.

### H8: Cut/Block Separator Hypothesis

Status: untested.

Claim: Strong pruning corresponds to small separators between the endpoint regions in the local base-clique transition graph or dependency DAG.

Motivation: If the endpoints lie in regions connected by few dependency channels, the pruning edge may constrain many local degrees of freedom.

Test plan:

- Compute articulation vertices and bridges in the undirected local dependency graph.
- Compute minimum cuts separating endpoint-containing cliques.
- Compare separator size and side dimensions with \(\log_2 |\Xi_e|\).

Decision: Keep as secondary. It may explain pruning strength but probably not exact counts.

## Rejected or weakened ideas

### Raw count from \(|L_e\setminus V_0|\)

The raw local tree size \(2^{|L_e\setminus V_0|}\) is only an upper bound. Base clique constraints already remove branches before the explicit pruning edge is tested.

### Unlabelled powers of the base-clique adjacency matrix

The sums of powers of the unlabelled adjacency matrix count paths between base cliques, but they do not preserve which vertex carries the dependency. This appears insufficient for exact local solution counts.

## Open questions

- Is the affine-code property true for generic DDGP instances, or is it an artifact of the current generator?
- Does the affine-code property hold for \(K>2\)?
- Does the labelled-violation rank formula continue to hold for \(K>3\)?
- Can the mirror-compatible mask formula be proved for generic one-pruning-edge DDGP instances?
- Is the 1409/1410 nongeneric failure fully explained by accidental mirror collinearity?
- How do multiple pruning edges interact: intersection of affine codes, rank drop, or non-affine behavior?

## Next tests

- Extend the generator and enumerator beyond \(K=3\), or rewrite the geometric enumerator using a general \(K\)-sphere intersection routine.
- Search for a proof that mirror-compatible graph masks generate the generic local stabilizer.
- Search for a proof that the labelled violation kernel gives the stabilizer for multiple pruning edges.
- Formalize the role of nongeneric mirror coincidences as exceptional algebraic cases.
